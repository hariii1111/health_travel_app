## 1. サービス概要
このサービスは、日々の運動量をカロリーから距離に変換し、日本を旅するように健康管理ができるアプリです。  
運動すればするほどキャラが成長し、ご当地アイテムを集めながら旅が進むため、楽しみながら運動習慣を継続できます。  
運動不足に悩む人が、ゲーム感覚で運動を続けられる仕組みを提供します。

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情
仕事や日常が忙しく、運動する機会がなかなか取れず、ジムに通っても長続きしない時期が続いていた。  
「運動しなきゃ」と思っていても、気づけば運動不足になっている自分に焦りやモヤモヤを感じていた。  
ただ目標を立てるだけでは続かず、楽しさや達成感がないと継続できないという実感があった。

### 2-2. なぜそれが「気になった」のか
自分はゲーム性や成長要素があるとモチベーションが上がり、楽しみながら継続できるタイプだと感じていた。  
「運動にもゲームのような仕組みがあれば続けられるのでは？」という思いが強くなり、  
運動不足を解消したい気持ちと、楽しさがないと続かない性格が結びつき、このアイデアが生まれた。

---

## 3. 課題の整理（Whyを3回繰り返して深掘り）

### 3-1. 表に見えている困りごと
- 運動を続けるモチベーションが保てない  
- 記録アプリは味気なくて続かない  
- 運動不足を自覚しているが行動に移せない  

### 3-2. なぜそれが問題なのか（Whyを3回繰り返す）

#### Why①：なぜ運動が続かないのか？
→ 運動しても「成果が実感できない」から。

#### Why②：なぜ成果が実感できないと困るのか？
→ 「やっても意味がない」と感じてしまい、行動が止まるから。

#### Why③：なぜそれがユーザーにとって深刻なのか？
→ 健康のために運動したいのに、続けられない自分に自己嫌悪を感じるから。

### 3-3. 本当に解決したい課題
**「運動の成果が実感できず、達成感が得られないために続けられない」という根本課題を解決したい。**

---

## 4. 想定ユーザーについて

### 4-1. 想定ユーザー（具体化）
**40代・デスクワーク中心の会社員（過去の自分）**

- 1日の大半をPCの前で過ごし、運動不足を自覚  
- 仕事終わりは疲れて動けず、休日は家で過ごしがち  
- 健康診断で「運動不足」「体重増加」を指摘され焦り  
- 記録アプリは味気なくて続かない  
- ゲームやキャラ育成が好きで、成長が見えると続けやすい  
- 「運動＝義務」ではなく「運動＝楽しみ」に変えたい  

### 4-2. 自分とユーザーの距離
→ 過去の自分そのもの。

### 4-3. 実際に使われる場面（具体化）
- **朝**：昨日の運動をまとめて記録し、旅が進んだことを確認  
- **昼休み**：10分歩いたのでその場で記録し、「あと80kmで次のエリア」と進捗を確認  
- **夜**：疲れて運動できなかった日でも旅マップを見てモチベ維持  
- **休日**：30分運動してキャラが成長、ご当地アイテムを獲得  

### 4-4. 導入・継続の流れ

#### ① どうやってこのサービスを知るのか（導入）
- 健康診断で運動不足を指摘されたタイミング  
- SNSで「運動が旅になるアプリ」として紹介されているのを見た  
- 友人から「ゲーム感覚で続く」と勧められた  

#### ② 最初に何をするのか（初回体験）
- プロフィール入力（年齢・体重など）  
- キャラクター選択（※MVPでは1体のみ）  
- 最初の目的地（例：東京 → 神奈川）を設定  
- チュートリアルで「運動 → 距離 → 旅 → 成長」の流れを体験  

#### ③ なぜ続くのか（継続理由）
- 運動すると旅が進むという“物語性”がある  
- キャラが成長し、努力が可視化される  
- ご当地アイテム収集でコレクション欲が刺激される  
- マップで進捗が見えるため、達成感が得られる

## 5. 既存サービス・競合調査（体験ベース）

### 5-1. 競合比較表

| サービス名 | できること | できないこと | 本アプリとの違い（体験の差） |
|------------|------------|--------------|------------------------------|
| ドラクエウォーク | 歩数で冒険、RPG、キャラ育成、ご当地要素 | 運動量の記録や健康データとの連動 | 歩くことが中心で「運動習慣の形成」が目的ではない |
| ポケモンGO | 位置情報ゲーム、収集、探索 | 運動量の可視化、継続的な健康管理 | 運動はゲームの副産物であり、健康目的ではない |
| Habitica | 習慣化×RPG、タスク達成で成長 | 運動に特化した体験、旅の概念 | 生活全体の習慣化が目的で、運動に特化していない |
| Nike Run Club | ラン記録、コーチング、分析 | ゲーム性、旅、キャラ成長 | 健康管理が中心で、楽しさや継続の仕掛けが弱い |

### 5-2. 差別化を一文で
**「運動量がそのまま“旅の進行”と“キャラ成長”に変わることで、運動がゲームの物語として続いていく体験」を提供する点が唯一無二です。**

## 7. MVPで実装する機能（明確化）

### 1. 運動記録機能
- 種目・時間を入力して運動を記録  
- カロリー計算を自動で実行  

### 2. カロリー → 距離変換
- 距離 = カロリー / 50  

### 3. 旅の進捗表示
- 累計距離に応じて現在地を表示  
- 100kmごとに次のエリアへ進行  

### 4. キャラクター成長（仕様改善済）
- **運動時間に比例して経験値を付与（例：1分＝0.5exp）**  
- 長時間運動でも分割運動でも公平な成長が可能  

### 5. マイページ
- 今日の運動量  
- 累計距離  
- 現在のエリア  
- キャラのレベル  

### 6. ログイン / 新規登録
- 自分のデータを継続管理できる

## 8. このアプリの懸念点とその対策

### 懸念①：ユーザーが飽きて継続しなくなる可能性
**リスク**  
- 同じ運動・同じ画面が続くとマンネリ化する  
- 旅の進行が遅いと達成感が薄れる  

**対策（具体的行動）**  
- MVP期間中にユーザーヒアリングを実施し、「飽きポイント」を特定  
- 進行速度の調整（距離換算の見直し）  
- 週替わりのミッションや小目標を追加する検証を行う  

---

### 懸念②：運動量の手入力が手間で離脱につながる
**リスク**  
- 毎回手入力だと面倒  
- 忘れると記録が途切れ、モチベが下がる  

**対策（具体的行動）**  
- MVP段階で「最も入力が面倒なポイント」をユーザーにヒアリング  
- 種目のプリセット化・直前入力の自動補完を検証  
- 将来的にヘルスケア連携（歩数・距離）を検討  

---

### 懸念③：キャラ画像がフリー素材だと世界観に統一感が出ない
**リスク**  
- フリー素材とオリジナルのギャップが出る  
- 世界観が弱く、愛着が湧きにくい  

**対策（具体的行動）**  
- MVPではキャラは1体のみ  
- 衣装チェンジはMVPでは実装しない  
- 成長段階（Lv1〜Lv5）のみ画像を用意  
- フリー素材で仮実装し、ユーザーの反応を見て本リリース時にオリジナル制作を検討  

---

### ④ 友達と旅レース
- 距離を競ったり、同じルートを進める  
- → 運動が“仲間と楽しむ体験”に変わる  

### ⑤ AI運動提案（本リリース）
- 運動履歴から最適な運動を提案  
- → 迷わず運動でき、継続しやすくなる

## 10. 運動記録の仕様（カロリー計算）

### カロリー計算式（MVP）
消費カロリー = METs × 60kg × (運動時間 / 60) × 1.05

### 種目ごとの METs（例）
| 種目 | METs |
|------|------|
| ウォーキング | 3.5 |
| ランニング | 7.0 |
| サイクリング | 6.0 |
| 筋トレ | 5.0 |
| ヨガ | 2.5 |
